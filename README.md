# بوابة إدارة الشركات والأسطول

نظام إدارة شركات وأسطول (موظفين، مركبات، عقود، طلبات توصيل، تحصيل مالي، تصدير Word) يعمل من صفحة واحدة `index.html` ويخزن البيانات سحابياً في **Google Firebase**.

---

## ربط الموقع مع Google Firebase (خطوة بخطوة)

الكود جاهز ومربوط برمجياً — يتبقى فقط تفعيل الخدمات من لوحة تحكم Firebase بحسابك. تستغرق العملية 5 دقائق تقريباً.

### 1) افتح مشروعك في Firebase

- ادخل على [console.firebase.google.com](https://console.firebase.google.com) بحساب Google الخاص بك.
- المشروع المربوط حالياً في الكود اسمه: **`datab-3af7f`**.
- إذا لا يظهر عندك هذا المشروع، أنشئ مشروعاً جديداً بزر **Add project** ثم انتقل للخطوة (4) لتحديث المفاتيح.

### 2) فعّل تسجيل الدخول بالبريد الإلكتروني

- من القائمة الجانبية: **Build ← Authentication**.
- اضغط **Get started** (إذا كانت أول مرة).
- تبويب **Sign-in method** ← اختر **Email/Password** ← فعّل الخيار الأول (Email/Password) ← **Save**.

> الموقع يحول اسم مستخدم الشركة تلقائياً إلى بريد بصيغة `username@saas-erp.com`، لذلك تفعيل Email/Password هو كل المطلوب.

### 3) أنشئ قاعدة بيانات Firestore وطبّق قواعد الحماية

- من القائمة الجانبية: **Build ← Firestore Database** ← اضغط **Create database**.
- اختر أقرب منطقة لك (مثلاً `eur3` أو `me-central1`) ← ابدأ بوضع **Production mode**.
- بعد الإنشاء افتح تبويب **Rules** والصق القواعد التالية ثم اضغط **Publish**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // كل شركة تقرأ وتكتب بياناتها فقط — لا يمكن لأي حساب آخر الوصول لها
    match /artifacts/{appId}/users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

> ⚠️ لا تستخدم القاعدة المفتوحة `allow read, write: if true;` — فهي تسمح لأي شخص على الإنترنت بقراءة وتعديل بيانات جميع الشركات. القواعد أعلاه تعزل بيانات كل شركة بحسابها.

### 4) (فقط إذا أنشأت مشروعاً جديداً) حدّث مفاتيح الاتصال

- من **Project settings** (أيقونة الترس) ← قسم **Your apps** ← أضف تطبيق ويب `</>` إن لم يوجد.
- انسخ كائن `firebaseConfig` الظاهر لك.
- افتح `index.html` وابحث عن `const firebaseConfig` (بداية السكربت) واستبدل القيم بقيمك.

### 5) أضف نطاق موقعك للنطاقات الموثوقة

- **Authentication ← Settings ← Authorized domains** ← **Add domain**.
- أضف النطاق الذي تستضيف عليه الموقع (مثلاً `username.github.io` إذا تستخدم GitHub Pages). النطاقان `localhost` و `*.firebaseapp.com` مضافان تلقائياً.

### 6) جرّب

افتح الموقع واضغط أحد أزرار **الحسابات التجريبية** (شركة الصقر / مؤسسة الخليج) — إذا دخلت للوحة التحكم فالربط شغال ✅. أي نقص في الإعداد سيظهر لك كرسالة خطأ عربية واضحة أعلى شاشة الدخول تخبرك بالخطوة الناقصة بالضبط.

## إضافة حساب شركة جديدة (يدوياً)

التسجيل الذاتي من الموقع معطّل عمداً — إضافة الشركات تتم من لوحة Firebase فقط:

1. **Authentication ← Users ← Add user**.
2. البريد الإلكتروني: اكتب اسم المستخدم متبوعاً بـ `@saas-erp.com` — مثال: لشركة باسم مستخدم `elite` اكتب `elite@saas-erp.com`.
3. كلمة المرور: 6 خانات أو أكثر.
4. سلّم صاحب الشركة اسم المستخدم (`elite` فقط، بدون البريد) وكلمة المرور — يدخل بهما من شاشة تسجيل الدخول مباشرة، وستكون بياناته معزولة تلقائياً عن باقي الشركات.
